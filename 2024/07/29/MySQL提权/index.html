数据库权限获取

数据库操作权限

也就是拿到MySQL的用户名和密码，主要包括下面四种：

  1. MySQL 3306 端口弱口令爆破
  2. sqlmap 注入的 --sql-shell 模式

  4. CVE-2012-2122 等这类漏洞直接拿下 MySQL 权限

Webshell权限

into outfile写shell

什么是into outfile？

在 Shell 脚本中，into outfile 是用于将命令的输出重定向到文件的一种方式

具体来说，重定向运算符 > 和 >> 用于将命令输出写入文件。

  • >：将命令输出写入指定文件。如果文件已存在，则会覆盖文件内容

  • >>：将命令输出追加到指定文件的末尾。如果文件不存在，则会创建文件

    这些方法都用于在 Shell 脚本中处理命令输出，使得脚本可以将信息存储在文件中以供后续使用或分析。

使用条件：

  • 知道网站物理路径
  • 高权限数据库用户
  • load_file () 开启 即 secure_file_priv 无限制
  • 网站路径有写入权限

首先基础语法查询是否 secure_file_priv 没有限制

Value 说明
NULL 不允许导入或导出
/tmp 只允许在 /tmp 目录导入导出
空 不限制目录

在 MySQL 5.5 之前 secure_file_priv 默认是空，这个情况下可以向任意绝对路径写文件

在 MySQL 5.5 之后 secure_file_priv 默认是 NULL，这个情况下不可以写文件

UDF提权

概述

UDF用户自定义函数（user defined function），它是用户创建的函数，可用于扩展MySQL的权限，通常用于内置函数不能满足需求的情况下，通过用户自定义函数，可以达到操作系统主机的效果，也就是所谓的MySQL提权

思路：

自定义的函数可以实现各种功能，比如getshell,如果数据库里能有我们自定义的函数，就可以实现对服务器的操控。

步骤：

  1. 获得动态链接库
  2. 寻找插件目录
  3. 写入动态链接库
  4. 创建自定义函数并调用命令

前提条件

UDF提权的前提条件：

1.必须是root权限（需要创建和抛弃自定义函数）

2.secure_file_priv=空（必须为空，secure_file_priv为null或者为/tmp/都不行，因为它需要在指定的位置写入udf文件）

在这里碰到一个问题，就是没有写入权限

ERROR 1 (HY000): Can't create/write to file '/usr/lib/mysql/plugin/udf.so' (Errcode: 13 "Permission denied") MariaDB [(none)]> MariaDB 无法在 /usr/lib/mysql/plugin/ 目录中创建或写入文件，原因是权限不足

解决方法：

用 chown 和 chmod 命令来更改目录的所有权和权限

可以通过以下语句来查询 MariaDB 用户（通常是 mysql）对该目录有没有写入权限：

  • 修改一下 允许mysql用户写入（多此一举）
  • chown -R mysql:mysql /usr/lib/mysql/plugin/
